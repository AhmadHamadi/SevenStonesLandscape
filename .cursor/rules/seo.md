# SEO — Always use project SEO skills

When the user asks to improve SEO, audit the site, add meta tags, fix rankings, do keyword research, build location/template pages, or anything related to search visibility:

1. **Read the relevant skill(s)** from `.cursor/skills/` before implementing:
   - **seo** — Technical + on-page SEO, structured data, sitemap, robots, canonicals, audit checklist.
   - **roier-seo** — Lighthouse/PageSpeed audits and auto-fix patterns (meta, schema, Core Web Vitals, accessibility).
   - **programmatic-seo** — Pages at scale (location pages, comparisons, directory, templates); unique value per page, URL structure, internal linking.
   - **seo-optimizer** — Keyword strategy, content SEO, E-E-A-T, topic clusters, featured snippets, local SEO, monitoring.

2. **Apply their patterns** (title/meta length, heading hierarchy, JSON-LD schemas, URL rules, image alt, internal links) rather than ad-hoc changes.

3. For **SEVENSTONES** specifically: local business (Hamilton/Burlington/Oakville), so prioritize LocalBusiness schema, location keywords, and the programmatic-seo playbooks for "[service] in [city]" if we scale pages.

Trigger keywords: SEO, meta tags, sitemap, structured data, schema, audit, Lighthouse, PageSpeed, keyword, ranking, programmatic SEO, location pages, template pages.
